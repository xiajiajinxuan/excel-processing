# 食堂扣缴规则说明文档

## 📋 规则概述

**规则名称**：食堂扣缴规则  
**规则ID**：`canteen_deduction_rules`  
**版本**：1.0  
**作者**：系统  
**功能描述**：处理员工食堂消费记录和打卡记录，自动计算员工的就餐减免次数、实际就餐次数、实际就餐金额、应扣就餐减免次数和应扣金额，并生成详细的扣缴记录和月度汇总。

## 📥 输入要求

### Excel文件结构

处理文件必须是一个包含两个工作表的Excel文件（`.xlsx`格式）：

#### 1. 打卡记录工作表

**工作表名称**：`打卡记录`

**必需列**：
- `姓名`：员工姓名（文本类型）
- `工号`：员工工号（文本或数字类型）
- `日期`：打卡日期（日期类型，格式：YYYY-MM-DD）
- `实际出勤工时`：当日实际出勤工时数（数字类型，单位：小时）

**示例数据**：

| 姓名 | 工号 | 日期       | 实际出勤工时 |
|------|------|------------|--------------|
| 张三 | 001  | 2024-01-15 | 8.0          |
| 李四 | 002  | 2024-01-15 | 6.5          |
| 王五 | 003  | 2024-01-15 | 4.0          |

#### 2. 消费记录工作表

**工作表名称**：`消费记录`

**必需列**：
- `工号`：员工工号（文本或数字类型，需与打卡记录中的工号匹配）
- `姓名`：员工姓名（文本类型）
- `交易金额`：交易金额（数字类型，单位：元）
- `交易日期`：交易日期（日期类型，格式：YYYY-MM-DD）
- `餐别`：餐别类型（文本类型，可选值：`早餐`、`午餐`、`晚餐`等）

**示例数据**：

| 工号 | 姓名 | 交易金额 | 交易日期   | 餐别 |
|------|------|----------|------------|------|
| 001  | 张三 | 7.0      | 2024-01-15 | 午餐 |
| 001  | 张三 | 7.0      | 2024-01-15 | 晚餐 |
| 002  | 李四 | 4.3      | 2024-01-15 | 早餐 |

### 数据验证

规则会自动进行以下验证：

1. ✅ 检查Excel文件是否包含必需的工作表
2. ✅ 检查每个工作表是否包含必需的列
3. ✅ 验证日期格式是否正确
4. ✅ 检查打卡记录是否为空
5. ✅ 删除无效日期的记录

## 🧮 计算逻辑详解

### 1. 享受就餐减免次数计算

根据员工的实际出勤工时，计算当日可享受的就餐减免次数。

**计算公式**：

```
如果 实际出勤工时 < 4小时：
    享受就餐减免次数 = 0次
否则如果 实际出勤工时 < 8小时：
    享受就餐减免次数 = 1次
否则（实际出勤工时 ≥ 8小时）：
    享受就餐减免次数 = 2次
```

**计算示例**：

| 实际出勤工时 | 享受就餐减免次数 | 说明 |
|--------------|------------------|------|
| 3.5小时      | 0次              | 不足4小时，不享受减免 |
| 6.5小时      | 1次              | 4-8小时之间，享受1次减免 |
| 8.0小时      | 2次              | 8小时及以上，享受2次减免 |
| 10.0小时     | 2次              | 8小时及以上，享受2次减免 |

### 2. 实际就餐次数计算

根据消费记录工作表，统计员工在打卡日期当天的实际就餐次数。

**计算逻辑**：

1. 遍历打卡记录中的每一行
2. 根据`工号`和`日期`，在消费记录中查找匹配的记录
3. 匹配条件：
   - `消费记录.工号` == `打卡记录.工号`
   - `消费记录.交易日期` == `打卡记录.日期`（按日期匹配，忽略时间）
4. 匹配到的记录数量即为该员工当日的实际就餐次数

**计算示例**：

假设打卡记录中有一条记录：
- 工号：001
- 日期：2024-01-15

消费记录中有以下匹配记录：
- 工号001，日期2024-01-15，餐别：早餐
- 工号001，日期2024-01-15，餐别：午餐
- 工号001，日期2024-01-15，餐别：晚餐

则实际就餐次数 = 3次

### 3. 实际就餐金额计算

根据消费记录中的餐别，计算员工当日的实际就餐金额。

**餐别价格标准**：

- **早餐**：4.3元/次
- **午餐/晚餐**：7.0元/次（除早餐外的其他餐别）

**计算逻辑**：

1. 对于每个匹配的消费记录，根据`餐别`确定单价
2. 如果`餐别` == `早餐`，则单价 = 4.3元
3. 否则，单价 = 7.0元
4. 累加所有匹配记录的金额，得到实际就餐金额
5. 结果保留2位小数

**计算公式**：

```
实际就餐金额 = Σ(餐别单价)
其中：
  如果 餐别 == "早餐"：
      餐别单价 = 4.3
  否则：
      餐别单价 = 7.0
```

**计算示例**：

假设某员工在2024-01-15的消费记录：
- 早餐：1次 → 金额 = 4.3元
- 午餐：1次 → 金额 = 7.0元
- 晚餐：1次 → 金额 = 7.0元

实际就餐金额 = 4.3 + 7.0 + 7.0 = 18.3元

### 4. 消费明细生成

生成格式化的消费明细字符串，用于记录每餐的消费情况。

**格式**：`餐别:金额;餐别:金额;...`

**示例**：
- `早餐:4.3;午餐:7.0;晚餐:7.0`
- `午餐:7.0;晚餐:7.0`
- `早餐:4.3`

### 5. 应扣就餐减免次数计算

计算员工超出享受减免次数的就餐次数，即需要扣缴的减免次数。

**计算公式**：

```
应扣就餐减免次数 = 实际就餐次数 - 享受就餐减免次数

如果 应扣就餐减免次数 < 0：
    应扣就餐减免次数 = 0（不扣缴，取最大值0）
```

**计算示例**：

| 实际就餐次数 | 享受就餐减免次数 | 应扣就餐减免次数 | 说明 |
|--------------|------------------|------------------|------|
| 3次          | 2次              | 1次              | 超出1次，需扣缴1次 |
| 2次          | 2次              | 0次              | 正好，无需扣缴 |
| 1次          | 2次              | 0次              | 未超出，无需扣缴 |
| 5次          | 1次              | 4次              | 超出4次，需扣缴4次 |

### 6. 应扣金额计算

根据实际就餐金额和应扣就餐减免次数，计算需要扣缴的金额。

**计算公式**：

```
如果 实际就餐次数 == 0：
    应扣金额 = 0
否则：
    平均餐费 = 实际就餐金额 / 实际就餐次数
    应扣金额 = 平均餐费 × 应扣就餐减免次数
    应扣金额 = round(应扣金额, 2)  # 保留2位小数
```

**计算逻辑说明**：

1. 如果员工当日没有实际就餐，则无需扣缴
2. 计算平均餐费：用实际就餐总金额除以实际就餐次数
3. 应扣金额 = 平均餐费 × 应扣就餐减免次数
4. 结果四舍五入到2位小数

**计算示例**：

**示例1**：
- 实际就餐次数：3次
- 实际就餐金额：18.3元
- 享受就餐减免次数：2次
- 应扣就餐减免次数：1次

计算过程：
- 平均餐费 = 18.3 / 3 = 6.1元
- 应扣金额 = 6.1 × 1 = 6.1元

**示例2**：
- 实际就餐次数：5次
- 实际就餐金额：32.3元（早餐1次4.3元 + 午餐2次14元 + 晚餐2次14元）
- 享受就餐减免次数：1次
- 应扣就餐减免次数：4次

计算过程：
- 平均餐费 = 32.3 / 5 = 6.46元
- 应扣金额 = 6.46 × 4 = 25.84元

### 7. 月度汇总计算

按员工工号和年月进行分组汇总，生成月度统计数据。

**汇总字段**：

- `姓名`：取该员工该月的最后一个姓名（通常姓名不变）
- `工号`：员工工号
- `年月`：年月字符串（格式：YYYY-MM）
- `享受就餐减免次数`：该月所有日期的享受就餐减免次数之和
- `实际就餐次数`：该月所有日期的实际就餐次数之和
- `实际就餐金额`：该月所有日期的实际就餐金额之和（保留2位小数）
- `应扣就餐减免次数`：该月所有日期的应扣就餐减免次数之和
- `应扣金额`：该月所有日期的应扣金额之和（保留2位小数）

**汇总公式**：

```
月度汇总 = GROUP BY (工号, 年月) {
    姓名 = 取最后一个值
    享受就餐减免次数 = SUM(每日享受就餐减免次数)
    实际就餐次数 = SUM(每日实际就餐次数)
    实际就餐金额 = SUM(每日实际就餐金额)
    应扣就餐减免次数 = SUM(每日应扣就餐减免次数)
    应扣金额 = SUM(每日应扣金额)
}
```

## ⚠️ 注意事项

### 数据要求

1. **日期格式**：日期列必须能够被正确解析为日期类型，支持常见日期格式（YYYY-MM-DD、YYYY/MM/DD等）
2. **工号匹配**：消费记录中的工号必须与打卡记录中的工号完全匹配（区分大小写）
3. **日期匹配**：消费记录的日期必须与打卡记录的日期完全匹配（按日期匹配，忽略时间部分）
4. **餐别识别**：餐别列中，只有值为`早餐`的记录会被识别为早餐，其他值（如`午餐`、`晚餐`、`中餐`等）均按7.0元计算

### 计算规则说明

1. **享受就餐减免次数**：基于实际出勤工时计算，不依赖实际就餐情况
2. **实际就餐次数**：基于消费记录统计，如果某天没有消费记录，则实际就餐次数为0
3. **应扣金额计算**：使用平均餐费计算，确保扣缴金额的公平性
4. **金额精度**：所有金额字段保留2位小数，采用四舍五入方式

### 错误处理

规则会自动处理以下情况：

1. ✅ **无效日期**：自动删除无法解析的日期记录
2. ✅ **缺失列**：如果缺少必需列，会返回明确的错误信息
3. ✅ **空数据**：如果打卡记录为空，会返回错误信息
4. ✅ **工作表缺失**：如果缺少必需的工作表，会返回错误信息

### 性能考虑

1. **大数据量**：对于包含大量记录的文件（>10000行），处理时间可能较长，请耐心等待
2. **内存使用**：处理大文件时，会占用较多内存，建议确保有足够的可用内存
3. **文件大小**：建议处理的Excel文件大小不超过50MB

## 🔧 技术实现细节

### 数据处理流程

1. **文件读取**：使用`pandas.read_excel()`读取Excel文件
2. **数据验证**：检查必需列和工作表是否存在
3. **日期转换**：将日期列转换为pandas的datetime类型
4. **数据匹配**：使用pandas的布尔索引进行数据匹配
5. **计算处理**：逐行计算各项指标
6. **结果汇总**：使用`groupby()`进行月度汇总
7. **文件输出**：使用`pandas.ExcelWriter()`写入Excel文件

### 关键函数

- `calculate_meal_exemption(hours)`：计算享受就餐减免次数
- `calculate_deduction_amount(row)`：计算应扣金额
- `process(data_df, **kwargs)`：主处理函数
